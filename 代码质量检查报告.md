## 一、后端 Python 代码质量

### 1.1 异常处理问题 🔴 高优先级

**裸异常（Bare except）**：共发现 **11 处** `except: pass` 或单行裸 except，会静默吞掉所有异常，极难排查问题。

| 文件 | 行号 | 问题描述 |
|------|------|---------|
| `app/api/im_gateway.py` | 343 | `except: pass` — 吞掉所有异常 |
| `app/api/im_gateway.py` | 400 | `except: return JSONResponse(...)` — 裸 except |
| `app/api/im_gateway.py` | 418 | `except: return {"status": "error"}` — 裸 except |
| `app/api/admin.py` | 445, 454, 471, 474, 477 | 批量 `except: c.xxx = []` — 裸 except |
| `app/services/cj_service.py` | 129, 148 | `except: pass` — 静默忽略异常 |
| `app/core/celery_instrumentation.py` | 64-65 | `except: pass` — 丢弃监控指标错误 |

**过宽异常捕获**：`except Exception` 共 **37 处**，多数没有适当的重新抛出，建议捕获更具体的异常类型。

**改进建议**：
```python
# ❌ 错误做法
except: pass
except Exception: pass

# ✅ 正确做法
except (ValueError, KeyError) as e:
    logger.warning("Specific error: %s", e)
    raise  # 或返回合适的错误响应
```

### 1.2 安全问题 🔴 高优先级

#### 弱默认密码
**文件**：`app/core/config.py:17`
```python
POSTGRES_PASSWORD: str = os.getenv("POSTGRES_PASSWORD", "password")
```
默认密码为 `"password"`，若环境变量未设置，将以弱密码连接数据库。

**建议**：将默认值改为 `""` 并在 `model_post_init` 中针对生产环境强制校验。

#### 根目录 .env 中存在 ngrok 临时地址
**文件**：`/.env:28`
```
BACKEND_URL=https://35c5-38-175-103-191.ngrok-free.app
```
根目录 `.env` 疑为调试遗留，包含临时 ngrok 隧道地址，应清理或替换为正式域名。

#### EXA_ADMIN_KEY 明文存储于根目录 .env
**文件**：`/.env`
```
EXA_ADMIN_KEY=26c6b381-2736-4513-890d-d71998f0b586
```
Admin 级别密钥直接明文存储，存在泄露风险。

#### byokStorage 本地混淆强度不足
**文件**：`frontend/src/services/byokStorage.ts:14`
```typescript
const SALT = "0buck_local_salt_8f9a2b";  // 硬编码盐值
```
用户自带的 API Key 仅用 Base64 + 固定盐做混淆，防护极弱。代码注释本身也指出了这个问题，建议使用 Web Crypto API。
