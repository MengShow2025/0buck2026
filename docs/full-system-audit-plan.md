# 0Buck 全项目功能图谱与测试清单 (Full System Audit Plan)

> **目标**：在经历代码丢失与架构重构后，对 0Buck 项目进行一次地毯式的全量回归测试，确保没有任何隐性功能被破坏。本清单将项目拆分为可独立验证的模块，便于逐一确认。

---

## 模块一：📱 前端 VCC 交互架构 (Vortex Chat Container)

### 1.1 全局框架与响应式
- [ ] **开屏与遮罩**：SplashScreen 动画是否正常展现？遮罩层是否严格限制在手机框内（不溢出到大屏幕）？
- [ ] **主题切换**：点击暗色/亮色切换，背景、文字颜色、阴影是否平滑过渡？
- [ ] **多语言切换**：点击语言切换，UI 界面词条是否实时更新？`t()` 函数翻译是否生效？
- [ ] **大屏适配**：在桌面端（≥1024px），界面是否正确居中呈现“手机框”且带有发光网格背景？窗口缩放时是否会发生顶部裁切？
- [ ] **品牌 Logo**：顶部 Header 是否使用了统一的 `ob-wordmark` 样式且无异常拉伸？

### 1.2 沙龙与通讯录 (Lounge & Contacts)
- [ ] **双 Tab 切换**：沙龙顶部的“聊天 (Chat)”和“发现 (Discover)”Tab 切换是否顺畅？下划线动画是否跟随？
- [ ] **+ 号菜单**：点击右上角 + 号，下拉菜单排序是否为“1. 添加好友、2. 发起群聊、3. 扫一扫”？
- [ ] **通讯录管理**：联系人列表（ContactsDrawer）能否正常打开？
- [ ] **好友申请**：“新的朋友（NewFriendsDrawer）”抽屉能否打开？同意/忽略操作后，状态是否持久化存入 `localStorage`，刷新不丢失？
- [ ] **黑名单联动**：在联系人列表中点击“拉黑”，好友是否立刻从列表消失并进入“黑名单（BlacklistDrawer）”？在黑名单中“解除拉黑”，好友是否能立刻恢复？

### 1.3 核心聊天与 AI 管家 (ChatRoom)
- [ ] **群聊菜单**：在群聊中点击右上角 `...`，是否弹出“成员管理”、“退群”、“关群”菜单？
- [ ] **退群/关群隐藏**：点击“退群”后，该群是否立即从沙龙会话列表中消失？
- [ ] **隐藏群恢复**：沙龙列表底部是否出现“恢复隐藏群”按钮？点击后群组是否重新出现？
- [ ] **AI 系统指令**：输入特定语言（如“打开分享”），AI 是否能下发系统动作（System Action）如 `OPEN_SHARE_MENU`，并成功唤起对应的抽屉？

### 1.4 推广分享与裂变 (ShareDrawer)
- [ ] **抽屉唤起**：点击分享按钮能否正常滑出 ShareDrawer？
- [ ] **奖励面板展示**：顶部是否正确拉取并显示了当前用户的身份、Tier、分销比例和粉丝奖励比例？
- [ ] **悬停动态提示**：鼠标 Hover 到卡片类型（如 Invite Card）或平台图标（如 WhatsApp）上时，底部的蓝色说明区是否会动态切换对应的文案？
- [ ] **分享生成**：选择平台并点击生成卡片，是否能模拟生成分享链接/模板？

## 模块二：🛒 无头电商与核心交易链路

### 2.1 商品展示与浏览
- [ ] **发现流 (Prime)**：商品列表是否正常拉取（支持分页/无限滚动）？
- [ ] **商品详情页 (ProductDetail)**：点击商品能否正常弹出详情抽屉？价格、图集、富文本描述渲染是否正常？
- [ ] **多平台比价**：是否展示了与 Amazon/eBay 等平台的价格对比（Compare at Price）和划线价？

### 2.2 购物车与结算
- [ ] **加购操作**：选择规格（Variant）后加入购物车，购物车角标是否更新？
- [ ] **结算页 (CheckoutDrawer)**：能否正常填写地址？
- [ ] **优惠叠加逻辑**：测试优惠券“多选”功能，是否正确执行了“互斥/可叠加”的前置判定？（不可叠加的券是否被禁用或提示错误？）
- [ ] **运费与税费**：总价计算是否包含了运费（如亚马逊运费标准）和相关的缓冲成本（Buffer Cost）？

### 2.3 支付与回调闭环
- [ ] **防抓包篡改**：前端提交的金额在后端是否有二次重算和强校验？
- [ ] **Shopify 唤起**：能否正确获取 Checkout URL 并唤起 Shopify 支付模态框？
- [ ] **订单状态流转**：模拟 Shopify Webhook 回调（如 `orders/paid`），后端防重队列（Truth Engine Queue / DB 级防重）是否正常处理？

### 2.4 特殊玩法 (互斥规则测试)
- [ ] **20期签到返现**：购买带有“100% Back”标签的商品后，是否在 Cashback 模块生成了 pending 状态的返现记录？
- [ ] **拼团免单互斥**：触发“拼团免单”后，该商品的“签到返现”资格是否自动作废（反之亦然）？前端订单中心是否有清晰的互斥作废提示？

## 模块三：⚙️ Admin 管理后台 (Admin Dashboard)

### 3.1 商品审核流 (Truth Engine Queue)
- [ ] **审核列表**：管理后台 `/admin/products` 能否正常加载状态为 `pending` 的候选商品池？
- [ ] **审核抽屉**：点击审核，能否完整显示图库、中英双语信息、采购成本、售价利润率？
- [ ] **AI 润色接口**：点击“AI Re-polish”，能否正常调用默认大模型（如 MiniMax）重新生成符合“9点心理狙击”规范的英文文案？
- [ ] **保存并发布**：点击保存并发布后，数据能否正确同步至 Shopify，并在数据库中从 `candidate_products` 转移到 `products` 正式表？（需测试是否还会报 404/500 错误）。

### 3.2 仪表盘与配置
- [ ] **财务大盘**：核心报表（GMV、利润等）接口能否正常渲染？
- [ ] **AI 管家配置**：AIPersonaPage 能否正常编辑 L2 策略层系统模板？

## 模块四：🧠 后端引擎与底层数据流

### 4.1 数据入库与关联
- [ ] **字段完整性**：1688/CJ 抓取的数据入库时，`models/product.py` 中新加的字段（如 `source_pid`, `amazon_shipping_cost`）是否会丢失？
- [ ] **数据源解耦**：前端商品流是否已经完全从正式表 `products` 拉取，去除了 Mock 数据的干扰？

### 4.2 大模型与 API 安全
- [ ] **默认模型路由**：系统全局默认模型是否优先使用了 `MINIMAX_API_KEY`（绕过 Gemini 的 IP 限制）？
- [ ] **BYOK 安全隔离**：用户在前端输入的自定义 API Key 是否只保存在本地 `LocalStorage`，并且请求直接从前端发起（后端不中转不记录）？
- [ ] **并发与限流**：模拟多次高频调用，后端的 Key 池轮询和 429 降级逻辑是否生效？

## 测试执行规范建议

由于项目庞大，建议我们按如下顺序逐个击破：
1. 先测试 **[模块一：前端 VCC 交互架构]**，确保之前因代码丢失造成的 UI 框架问题彻底解决，用户操作不卡壳。
2. 接着测试 **[模块三：Admin 管理后台]**，打通“商品上架”的源头。有了真实的正式商品，前端才能买。
3. 然后测试 **[模块二：无头电商与交易链路]**，这是最核心的资金流，需要模拟完整的“浏览 -> 加购 -> 结算 -> Webhook 回调”闭环。
4. 最后检查 **[模块四：后端底层机制]** 的边界情况（如大模型限流、BYOK 安全）。
