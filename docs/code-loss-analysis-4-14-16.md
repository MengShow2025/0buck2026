# 代码丢失与功能退化分析报告
*(基于 `聊天记录 4:14-16.md` 与当前代码库的对比)*

## ✅ 幸存下来的功能（代码库中仍存在的）
1. **External IM 绑定体系**：设置页（`SettingsDrawer.tsx`）的飞书、WhatsApp、Telegram、Discord 四个图标网格，彩色/灰色的状态展示，以及后端的查询/解绑接口。
2. **推广卡片（Bot Promo Cards）主链路**：后端的发卡、链接解析（`from-link`）、固定专属卡（`my-fixed-invite`）接口；前端 `ShareDrawer.tsx` 的 4 种卡片（专属邀请、商品、商家、拼团）以及按平台发送的选项。
3. **GetStream 核心类**：`frontend/src/services/streamChat.ts` 客户端封装文件。

---

## ❌ 完全丢失或退化的功能（需重点恢复的灾区）

### 1. 推广分享页（ShareDrawer）的体验细节
*   **[丢失] 奖励规则与比例展示**：原本应拉取当前用户的“身份、等级、分销比例、粉丝奖励比例”展示在分享页。
*   **[丢失] 动态图文说明区**：原本实现了“鼠标悬停/点击不同平台或卡片时，下方动态切换使用说明及奖励规则”。
*   **[丢失] AI 快速唤起指令**：原本在 `App.tsx` 增加了 `OPEN_SHARE_MENU` / `OPEN_PROMO_SHARE` 等系统动作支持，以便 AI 语义唤起。

### 2. 沙龙（LoungeDrawer）的大分类架构
*   **[丢失] 微信式大分类 Tab**：当时确认的沙龙顶部应改为 **`聊天`** 和 **`发现`**（包含活动、广告、动态）两大 Tab，现在的 `LoungeDrawer.tsx` 退回了早期只有一个大列表的状态。
*   **[错位] 右上角 `+` 菜单排序**：当时明确要求顺序改为“1. 添加好友、2. 发起群聊、3. 扫一扫”。现在的代码里，顺序变回了发起群聊排在第一。

### 3. 微信式通讯录与好友管理
*   **[丢失] `NewFriendsDrawer.tsx` 文件**：当时做了一个独立的二级页“新的朋友”，用来处理同意/拒绝好友申请，现在该页面文件不存在，`GlobalDrawer.tsx` 的路由里也没有 `new_friends`。
*   **[丢失] 黑名单与拉黑功能**：`ContactsDrawer.tsx` 里新增的老友列表“拉黑”按钮，以及“黑名单管理（解除拉黑）”页面全部丢失。
*   **[丢失] 本地存储状态**：记录黑名单的 `vcc_blocked_friends` 等代码逻辑丢失。

### 4. 群聊管理与本地持久化闭环
*   **[丢失] 群管理菜单**：当时在 `ChatRoomDrawer.tsx` 右上角增加了“成员管理 / 退群 / 关群”操作。
*   **[丢失] 群组隐藏与恢复**：当时做好的“退群/关群后从会话列表隐藏（`vcc_left_groups` / `vcc_closed_groups`）”，以及沙龙底部的“一键恢复隐藏群”按钮。
*   **[丢失] 聊天记录纯本地化**：为了节省 GetStream 费用，当时特意修改的 `vcc_chat_messages_<chatId>` 纯本地存储约束逻辑脱落，`ChatRoomDrawer.tsx` 中也未引入 `streamChat.ts`。

### 5. 品牌 Logo 规范统一
*   **[丢失] 字体与样式全局应用**：聊天记录里提到替换了全局的 `index.html`（引入 Fira Code 字体）和 `VCCHeader.tsx`，定义了统一的 `ob-wordmark` 等 CSS 类。这些内容丢失导致了目前 UI（如开屏页）中 0Buck 标志显得巨大且不符合品牌规范。

## 💡 恢复建议与实施路径
针对上述丢失的内容，建议放弃零敲碎打的修补，按照以上 5 个专项，逐一进行**“精准还原”**：
1. **优先恢复 UI 与基础架构**：【品牌 Logo 规范统一】与【沙龙大分类架构】。
2. **恢复核心业务闭环**：【推广分享页的体验细节】。
3. **恢复关系链与聊天机制**：【微信式通讯录与好友管理】与【群聊管理与本地持久化】。
