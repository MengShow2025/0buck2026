  CRIT-01 — Gemini API Key 暴露给所有浏览器用户                              

  frontend/.env 中 VITE_GEMINI_API_KEY=AIzaSy... — Vite 会将所有 VITE_          
  前缀的变量打包进 JS bundle 并发给每个访客，任何人打开 DevTools
  就能拿到这个密钥。                                                            
                                                                             
  CRIT-02 — 加密主密钥使用公开默认值                                            
   
  config.py:44,61：                                                             
  MASTER_SECRET_KEY = os.getenv("MASTER_SECRET_KEY",                         
  "0buck_default_master_key_for_api_keys_32")       
  SECRET_KEY = os.getenv("SECRET_KEY",       
  "0buck_default_secret_key_for_sessions_32")
  .env 确认这个默认值在生产环境中实际被使用，任何人都能伪造 JWT 或解密用户数据。

  CRIT-03 — 真实第三方 API 密钥硬编码在源代码                                   
   
  config.py:106-129 — CJ、Mabang、BuckyDrop、YunExpress 的真实 API 密钥作为     
  Python 默认值写死，已进入 git 历史：                                       
  CJ_API_KEY: str = os.getenv("CJ_API_KEY", "2c0889e00f7a4bd1881989564c3e148c")
  BUCKYDROP_TEST_CODE: str = "87e05078db55..."  # 完全没有 os.getenv，永远硬编码
                                                                                
  CRIT-04 — Admin 密码明文比较（非 bcrypt）
                                                                                
  auth.py:86：
  if login_data.password == settings.DEFAULT_ADMIN_PASSWORD:  # 直接字符串比较  
  存在时序攻击漏洞，且 .env 中的 DEFAULT_ADMIN_PASSWORD=Lwjun1976 明文存储。 

  CRIT-05 — 5个金融端点完全无身份验证（IDOR）

  butler.py:112-149 — 以下端点没有 Depends(get_current_user)：
  - GET /profile/{user_id}
  - POST /settings/{user_id}                                                    
  - POST /points/redeem/{user_id} — 任何人可以消耗任意用户的积分
  - POST /checkin/{user_id} — 可以帮任意用户签到                                
  - POST /tools/call/{user_id} — 可以以任意用户身份调用 AI 工具

  auth.py:760,781, users.py:151：
  if val.primary_email_otp == "123456":   # 任何人都能改绑邮箱                  
  if rebind_in.new_email_otp != "654321":                                    

  CRIT-07 — OAuth 回调开放重定向

  auth.py:677-688：绝对 URL 未经验证直接用作重定向目标，攻击者可构造            
  ?redirect=https://evil.com 在登录后窃取会话。
                                                                                
  CRIT-08 — Shopify Webhook HMAC 验证在非生产环境被绕过                      

  webhooks.py 中，当 ENVIRONMENT != "production" 时，大多数 webhook
  直接跳过签名验证，伪造 webhook 即可触发退款/发货操作。
