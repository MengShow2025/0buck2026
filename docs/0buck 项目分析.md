0Buck 项目分析                                                              
                                                                                
  项目概述                                                                      
                                                                                
  0Buck 是一个 AI 驱动的无头电商平台，通过聊天界面聚合多个代购/dropshipping     
  供应链（1688、CJ、BuckyDrop 等），核心创新是 Persona OS —                     
  一个能学习用户偏好并管理复杂返利系统的三层 AI 管家。

  ---
  架构核心

  1. Persona OS（三层提示系统）

  L1 Enforcement（Admin 配置）→ 利润率强制、内容去标识化
  L2 Strategy（人格模板）→ cute_loli / rigorous_expert 等
  L3 Surface（用户自定义）→ 长期记忆 + 实时偏好
                                                                                
  2. LangGraph Agent 工具集
                                                                                
  - product_search, web_search, supply_library                               
  - order_status, coupons, UI actions
  - 失效切换：gemini-2.5-flash → gemini-2.0-pro

  3. BAP 协议（Business Attachment Protocol）

  聊天消息内嵌结构化 JSON 卡片：0B_PRODUCT_GRID、0B_CASHBACK_RADAR 等，前端     
  React 组件解析并渲染为富交互界面。
                                                                                
  4. 数据安全三区隔离                                                        

  - Zone 1：核心账务（AI 禁止访问）
  - Zone 2：用户交易数据
  - Zone 3：AI 交互/Shadow 数据（Shadow ID 映射，10年有效期）

  主要风险/观察                                                              
               
  1. AppContext.tsx 过大（146KB），违反文件聚焦原则（>800行），是潜在维护风险
  2. 返利系统复杂度极高（1,700行），业务逻辑密集，测试覆盖至关重要              
  3. 多 API 密钥依赖（Shopify、Google、Stream、1688等），.env 管理是安全关键点
  4. Async 反射学习循环（Reflection Loop）是创新亮点，但需要持续监控 AI 记忆质量
  5. 供应链去标识化是核心商业护城河，supply_chain.py 是最高优先级保护代码
