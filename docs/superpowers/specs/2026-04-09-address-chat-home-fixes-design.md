# 地址新增、聊天头部与首页断层修复设计

## 目标

- 在地址管理页内直接提供“新增地址”能力。
- 删除私聊、群聊抽屉页内部橙色头部上多余的返回箭头。
- 修复首页 AI 聊天页中部深浅断层，让该区域与下方聊天背景保持一致的深色视觉。

## 范围

- 修改 `frontend/src/components/VCC/Drawer/AddressDrawer.tsx`
- 修改 `frontend/src/components/VCC/Drawer/ChatRoomDrawer.tsx`
- 修改 `frontend/src/App.tsx`
- 修改 `frontend/src/components/VCC/VortexContainer.tsx`

## 方案

### 1. 地址新增

- 在现有地址抽屉内切换到新增地址表单，不增加二级抽屉。
- 字段包含：
  - 收件人
  - 手机号
  - 地址
  - 城市
  - 州/省
  - 邮编
  - 是否默认地址
- 保存后立即写入当前前端列表，并返回列表视图。

### 2. 私聊/群聊多余返回箭头

- 保留抽屉系统级返回/关闭能力。
- 删除聊天页内部橙色头部左侧多余的返回箭头按钮。
- 不修改聊天页其余头部信息布局与按钮逻辑。

### 3. 首页 AI 聊天页断层

- 首页中部发灰发浅的断层区域，统一改为和下方聊天区一致的深色背景。
- 只处理首页场景，不影响私聊、群聊抽屉页。
- 不通过改全局主题变量解决，避免再次污染其它页面。
- 首页背景层和消息流背景层需要保持单一深色来源，避免浅色根底透出。

## 实现边界

- 不修改私聊/群聊抽屉页当前正常的背景色和气泡样式。
- 不新增新的抽屉类型来处理地址新增。
- 不调整业务数据结构，只使用前端本地状态完成新增地址演示。

## 验收标准

- 地址页点击 `Add New Address` 后可填写并保存新地址。
- 私聊/群聊页仅保留一套抽屉导航控制，不再出现多余返回箭头。
- 首页截图中部原本发灰的断层区域变为深色，并与下方区域连贯。
- 不新增 TypeScript 或样式诊断错误。
