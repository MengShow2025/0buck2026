# 0Buck 奖励与分销系统全规格说明书 (Master Rewards Spec)

## 1. 系统概览
0Buck 奖励系统旨在通过高粘性（签到）、高爆发（拼团）和长尾收益（粉丝分销）构建增长闭环。系统包含六大核心支柱，并由一套**优先级引擎**驱动结算。

---

## 2. 六大核心支柱

### 支柱 1：签到返现 (Sign-in Cashback)
*   **触发条件**：订单确认收货且过退货期。
*   **周期**：500天内。
*   **结构**：分为 20 期发放，总额等于"奖励基数"（实付金额 - 运费 - 税费）。
*   **算法**：
    *   **P1-P2 (引导期)**：固定（4%, 6%），周期短（5d, 10d）。
    *   **P3-P20 (长期期)**：每期 25-30 天。包含常规期（3-5%）与爆点大奖期（8%, 10%）。
        *   **爆点出现机制（先小后大）**：采用分段隔离抽取算法。
            *   **P3-P10（前期）**：仅从普通奖励池抽取，爆点概率**比较低**。
            *   **P11-P20（后期）**：将剩余普通期与 8%、10% 两个爆点混合打乱，爆点出现概率**明显更高**。
        *   **权威算法参考（JavaScript）**：
            ```javascript
            /**
             * 0Buck 船长精简版生成器
             * 核心：P1-P2固定，P3-P20线性决定是否触发爆点池
             */
            const _ = require('lodash');

            function generate0BuckFinalRoadmap() {
                // 1. 初始 20 个组合全家桶
                const p1_p2 = [{ r: 4, d: 5 }, { r: 6, d: 10 }];
                const big_hits = [{ r: 8, d: 30 }, { r: 10, d: 30 }];
                const normal_30d = [
                    { r: 5, d: 30 }, { r: 5, d: 30 }, { r: 5, d: 30 }, { r: 5, d: 30 }, { r: 5, d: 30 }
                ];
                const normal_25d = [
                    { r: 5, d: 25 }, { r: 5, d: 25 }, { r: 5, d: 25 }, { r: 5, d: 25 },
                    { r: 3, d: 25 }, { r: 4, d: 25 }, { r: 4, d: 25 }, { r: 4, d: 25 },
                    { r: 4, d: 25 }, { r: 4, d: 25 }, { r: 4, d: 25 }
                ];

                // 2. 核心：分段隔离抽取
                // 前期 (P3-P10)：只从 16 个普通里抽 8 个，且优先排 30 天的磨损期
                let all_normals = _.shuffle([...normal_30d, ...normal_25d]);
                let early_stage = all_normals.slice(0, 8);
                let remaining_normals = all_normals.slice(8);

                // 后期 (P11-P20)：剩下的普通 + 两个大爆点，全部打乱
                let late_stage = _.shuffle([...remaining_normals, ...big_hits]);

                // 3. 最终流水线
                return [...p1_p2, ...early_stage, ...late_stage].map((item, index) => ({
                    phase: index + 1,
                    ratio: item.r,
                    days: item.d
                }));
            }
            ```
*   **断签规则**：
    *   任何一期中断，本期奖励作废。
    *   下次签到从**下一期**的第一天开始。
*   **续签机制**：每期可使用 1 次"续签卡"（积分兑换），最大断签天数需 ≤ 5天。

### 支柱 2：拼团购 (Group Buy / Share-to-Free)
*   **触发条件**：用户分享商品链接，吸引 3 人以上购买同款并确认收货。
*   **奖励额度**：每满 3 人返还 1 件对应商品的实付金额。
*   **抉择期**：收货后 5 天内（P1 完成前）可选。若开启签到发放，则失去此选项。
*   **优先级**：低于"达人分销"，高于"粉丝奖励"。

### 支柱 3：用户分销 (User Direct Distribution)
*   **触发条件**：通过普通用户分享的**商品专属链接**产生的一次性购买（与达人分销触发方式相同，均为商品专属链接）。
*   **奖励额度**：一次性结算订单基数的 **3% - 5%**（根据用户等级自动确定）。
*   **特性**：不限新老用户，只要是通过该特定商品链接购买即可触发。
*   **优先级**：仅次于达人分销。

### 支柱 4：用户邀请粉丝 (User Fan Invite)
*   **机制**：邀请新用户通过**固定专属注册链接**注册（与达人粉丝触发方式相同）。
*   **权益周期**：被邀请人注册后 **2 年内**的所有订单（非专属商品链接产生的订单）。
*   **奖励比例**：1.5% - 3%（基于邀请人等级，不同等级对应不同系数）。

### 支柱 5：达人分销 (KOL Distribution)
*   **门槛**：需提交申请并人工审核。
*   **触发条件**：通过达人生成的**商品专属链接**购买（与用户分销触发方式相同）。
*   **奖励比例**：8% - 20%（通过谈判确定，记录在达人 profile；根据达人等级系数不同）。
*   **优先级**：**最高级**。触发此奖励后，不计算拼团或粉丝奖励。

### 支柱 6：达人粉丝 (KOL Fan Invite)
*   **机制**：达人通过**固定专属注册链接**吸引新用户注册（与用户粉丝触发方式相同）。
*   **权益周期**：注册后 **2 年内**。
*   **奖励比例**：3% - 8%（根据达人等级系数不同）。
*   **结算**：用户确认收货过退货期后，自动发放到钱包。

## 3. 优先级引擎逻辑 (Priority Engine)
当一个订单涉及多个推荐关系或满足多个奖励条件时，系统按以下优先级 **仅发放一种** 奖励，且各档比例随用户/达人等级动态调整：

1.  **达人分销 (KOL Distribution)**: 8% - 20%（优先级最高）。
2.  **用户分销 (User Direct Distribution)**: 3% - 5%（优先级次之）。
3.  **拼团免单 (Group Buy)**: 3人免单奖励（由用户主动选择触发）。
4.  **达人粉丝奖励 (KOL Fan Rewards)**: 3% - 8%（2年LTV）。
5.  **用户粉丝奖励 (User Fan Rewards)**: 1.5% - 3%（2年LTV）。

*注：*
*   *同一笔订单只产生一种分销/推荐奖励。*
*   *用户自身的"签到返现"是基础权益，除非用户选择了"拼团免单"，此时二者互斥。*

## 4. 结算与提现
*   **资金池隔离**：系统实时计算预估奖励，但仅在"订单有效确认收货且过退货期"后转入可用余额。
*   **发放渠道**：平台内置钱包（Wallet）。钱包余额可用于**平台内支付**或**申请提现**。
*   **提现方式与手续费**：

    | 提现渠道 | 手续费 | 说明 |
    |:--------:|:------:|:----:|
    | **PayPal** | 3.2% + \$2 | 主流提现方式 |
    | **USDC（虚拟货币）** | \$2（固定） | 稳定币，链上到账 |

*   **最低提现金额**：**\$50**（所有渠道统一）。

## 5. 用户与达人等级系数
各奖励比例区间（如用户分销 3%-5%、KOL分销 8%-20%）均与**用户等级**或**达人等级**挂钩，等级越高系数越大。具体等级划分及对应系数由管理后台配置（非硬编码），可随运营策略动态调整。

## 6. 积分系统与续签卡 (Points & Renewal Card)

### 6.1 积分获取 (Daily Earning Caps)
为防止刷分并保持系统平衡，设立**每日非交易积分上限**：
*   **每日签到/微任务/评价总上限**：约 150 Pts。
    *   *说明：若用户仅靠签到和评价，至少需要连续活跃 20 天才能换取一张续签卡。*
*   **选品合伙人 (Wishlist Hub) 奖励**：
    *   **提交有效愿望清单**：+50 Pts (每次有效提交)。
    *   **建议被采纳 (成功招商/上架)**：+200 Pts 伯乐奖。
*   **无上限来源 (交易驱动)**：
    *   **自己购物**：按实付金额比例赠送。
    *   **邀请新用户下单**：成功邀请并下单获得大额积分。
    *   **粉丝购物**：绑定关系的粉丝下单，推荐人获得积分。

### 6.2 续签卡 (Renewal Card) 规则
*   **兑换价格**：**3000 Pts / 张** (固定整数)。
*   **使用限制**：
    *   **每期仅限使用 1 次** (对应 25-30 天的返现期)。
    *   **时效性**：断签后必须在 **5 天内** 使用，否则本期奖励永久作废。
    *   **有效期**：使用后续签本期进度，奖励不作废。

## 7. 财务风险预警与准备金 (Financial Risk Management)

为确保 500 天奖励系统的长期兑付能力，系统设立核心财务风控机制：

*   **奖励准备金池 (Reward Escrow Pool)**：每笔订单成交后，系统自动冻结对应比例的资金进入 Escrow 账户，专款专用，严禁挪作他用。
*   **风险水位线 (Reserve Ratio)**：
    *   **绿灯区 (Ratio > 1.2)**：准备金充足，奖励按常规比例释放。
    *   **黄灯预警 (Ratio < 1.1)**：触发风险预警，系统自动上调"汇率对冲层"（从 +0.5% 调至 +1.0%），以被动增加毛利收入。
    *   **红灯熔断 (Ratio < 1.0)**：暂缓大额非交易积分兑换，优先保障已购订单的 500 天返现兑付。
*   **汇率动态对冲**：系统每日自动对比中行汇率与实时采购成本，动态调整前端显示的本地货币价格，确保汇率波动损耗由 +0.5% 的 Buffer 完全覆盖。

## 8. 数据库模型建议
*   **Referrals表**：记录 `inviter_id`, `invitee_id`, `expiry_date`, `type (USER/KOL)`。
*   **RewardPhases表**：记录每个订单的 20 期执行状态。
*   **Points表**：记录用户积分，用于兑换"续签卡"。
*   **UserLevels表**：记录用户/达人等级及对应奖励系数，支持后台动态配置。
