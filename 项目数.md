# 0buck 核心项目结构树

```text
0buck/
├── backend/                       # 🐍 后端服务 (FastAPI)
│   ├── alembic/                   # 数据库迁移配置及脚本
│   ├── app/                       # 后端核心代码目录
│   │   ├── api/                   # API 路由层 (Auth, 业务接口)
│   │   ├── core/                  # 核心配置、依赖加载、安全认证
│   │   ├── db/                    # 数据库会话、连接池配置
│   │   ├── gateway/               # 第三方网关服务集成 (GetStream IM等)
│   │   ├── models/                # SQLAlchemy 数据模型定义
│   │   ├── schemas/               # Pydantic 数据验证与序列化模型
│   │   ├── services/              # 核心业务逻辑层 (OAuth, AI管家, 选品等)
│   │   ├── utils/                 # 工具类函数
│   │   ├── workers/               # Celery 异步任务及定时任务
│   │   └── tests/                 # 后端单元测试与集成测试代码
│   ├── data/                      # 后端本地测试数据和依赖配置
│   ├── prompts/                   # LLM (大语言模型) 的提示词模板库
│   ├── scripts/                   # 后端测试脚本与验证工具库 (含 API 冒烟测试等)
│   ├── requirements.txt           # 生产环境依赖
│   └── Dockerfile                 # 后端容器化构建文件
│
├── frontend/                      # ⚛️ 前端服务 (React + Vite + TypeScript)
│   ├── src/                       # 前端核心代码目录
│   │   ├── components/            # UI 组件库
│   │   │   └── VCC/               # 核心容器：Vortex Chat Container 相关组件
│   │   │       ├── Drawer/        # 所有抽屉组件 (Auth, Checkout, Settings等)
│   │   │       └── Desktop/       # 桌面端特定布局组件
│   │   ├── i18n/                  # 国际化多语言配置与词条
│   │   └── services/              # 前端网络请求及服务调用封装
│   ├── docs/                      # 前端相关的 QA 及检查清单文档
│   ├── scripts/                   # 前端工具脚本 (如多语言同步)
│   ├── tests/                     # 前端端到端 (E2E) 测试
│   ├── package.json               # 前端依赖配置
│   └── vite.config.ts             # Vite 打包构建配置
│
├── docs/                          # 📚 项目架构与技术规范文档
│   ├── 0Buck_v3.4_Vortex_Chat_Container_(VCC)_架构白皮书.md
│   ├── 0Buck_终极技术手册_v3.4.md
│   ├── 0Buck_AI_Butler_Master_Protocol.md  # AI 管家核心协议
│   ├── 0Buck_Database_Technical_Spec.md    # 数据库技术说明
│   ├── project-readiness-matrix.md         # 项目准备度与进度矩阵
│   ├── data-contracts/            # 数据结构契约文档
│   └── superpowers/               # Superpowers 工作规范及执行计划存放区
│
├── scripts/                       # 🛠️ 独立自动化与运维工具脚本
│   ├── check_*.py                 # 各种状态检测脚本
│   ├── sync_production_db.py      # 生产数据库同步
│   ├── generate_merchant_report_ultra.py # 报告生成工具
│   └── nuke_db.py / nuke_shopify.py # 环境重置及清理脚本
│
├── data/                          # 📦 数据抓取与预处理存放区
│   ├── 1688/                      # 1688 选品数据抽取脚本及 JSON 数据
│   └── data_v8_5_universal.json   # 核心静态数据包
│
├── ops/                           # ⚙️ 运维与部署配置
│   ├── db/                        # 数据库迁移记录
│   ├── env/                       # 多环境配置模板 (Staging / Production)
│   ├── prometheus/                # 监控及告警配置 (0buck-alerts.yml)
│   ├── release/                   # 发布检查清单与验证脚本
│   └── slo/                       # 服务等级目标 (SLO) 定义
│
├── reports/                       # 📊 业务与市场报告
│   └── 0Buck_Daily_Trends_*.md    # 每日趋势抓取与市场分析报告
│
├── deliverables/                  # 📤 交付物与导出数据
│   └── mirror_reports/            # 生成的镜像报告 (HTML / JSON)
│
├── package.json                   # 根目录全局配置 (可能用于部分工程化脚本)
└── docker-compose.yml             # 本地服务编排 (DB / Redis 等依赖)
```

**💡 架构概览总结：**
- **前后端分离**：标准的 `backend` (Python/FastAPI) 与 `frontend` (TS/React) 隔离架构。
- **重度业务脚本化**：根目录的 `scripts/` 与 `data/` 目录包含了大量的业务自动化处理脚本，涵盖了从数据抓取 (如 1688)、选品分析到数据库重置的各个环节。
- **高规范性文档**：`docs/` 中沉淀了极为详尽的白皮书和验收标准，尤其是结合了特有的 `VCC` (Vortex Chat Container) 前端架构和 `Superpowers` 工作机制。
