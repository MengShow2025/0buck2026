- **API 版本前缀**：`/api/v1`（见 [config.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/core/config.py#L5-L8) 与 [main.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/main.py#L38-L43)）

## 2. 接口字典（所有后端路由 / 输入输出）

### 2.1 Core API（api_router）

来源：[main.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/main.py#L38-L85)

#### GET `/api/v1/`
- **用途**：健康检查/欢迎信息
- **请求**：无
- **响应**：`{"message": "Welcome to ... API"}`

#### POST `/api/v1/sync/1688/{product_id}`
- **用途**：同步单个 1688 商品到本地 DB，并推送到 Shopify
- **Path 参数**：`product_id: str`
- **主要副作用**
  - DB：写入/更新 `suppliers`、`products`
  - Shopify：创建/更新 Product、写入 metafields（见下文 Shopify 规范）
- **响应**：
  - `{"status":"success","product":"<title_en>","shopify_id":"<shopify_product_id>"}`（见 [main.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/main.py#L45-L57)）

#### GET `/api/v1/users/{customer_id}`
- **用途**：查询用户钱包摘要与等级（简版 profile）
- **Path 参数**：`customer_id: str`（内部会 `int(customer_id)`）
- **响应**：
  - `{"id": "...", "name":"User ...", "wallet_balance": <available>, "level": <level>}`（见 [main.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/main.py#L59-L70)）
- **依赖**：`RewardsService.get_wallet_summary/get_user_level`

#### GET `/api/v1/customer/sync/{customer_id}`
- **用途**：强制同步某客户本地数据到 Shopify（用于“余额/等级 100% 准确”场景）
- **Path 参数**：`customer_id: str`（内部会 `int(customer_id)`）
- **响应**：
  - 成功：`{"status":"success","message":"Data synced for customer ..."}`
  - 失败：`{"status":"failed","message":"Failed to sync data to Shopify. Check permissions."}`
- **注意（实现缺口）**：此接口调用 `RewardsService.sync_customer_data_to_shopify`（见 [main.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/main.py#L72-L84)），但当前代码库中未找到该方法实现，属于“声明存在但不可用”的链路。

### 2.2 Agent API（LangGraph/Gemini 代理）

来源：[api/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/agent.py) + schema：[schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py)

这些接口同时提供契约别名路径：`/api/v1/agent/*`（等价于 `/api/v1/*`）。

#### POST `/api/v1/chat`
- **用途**：核心 AI 对话入口（非流式）
- **请求 Body**：`ChatRequest`
  - `content: string`
  - `session_id?: string`（默认 `"default"`）
  - `image_url?: string`
  - `locale?: string`（默认 `"en"`）
  - `currency?: string`（默认 `"USD"`）
  - 见 [schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py#L5-L11)
- **响应 Body**：`ChatResponse`
  - `id: string`、`role: string`、`content: string`
  - `type: "text" \| "products" \| "order_status"`
  - `products?: Array<object>`、`order_info?: object`
  - 见 [schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py#L12-L19)
- **依赖**：`agent_executor.ainvoke(...)`（[api/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/agent.py#L10-L16)）
- **实现说明**：请求会被组装为 LangGraph state（含 `locale/currency`）并使用 `thread_id=session_id` 保持会话连续性（见 [agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/agent.py)）。

#### POST `/api/v1/session`
- **用途**：创建持久化会话（写入 `ai_sessions`），并尝试为 Stream Chat 生成 token
- **请求 Body**：`SessionCreate`：`{ "user_id": "..." }`（[schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py#L21-L22)）
- **响应 Body**：`SessionResponse`：`{session_id,user_id,chat_token?,chat_api_key?,created_at}`（[schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py#L24-L30)）
- **DB 写入**：`ai_sessions(session_id,user_id,metadata_json,created_at,updated_at)`（见 [api/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/agent.py#L74-L83) 与 [ledger.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/models/ledger.py#L94-L101)）

#### POST `/api/v1/product_search`
- **用途**：独立的语义/视觉搜品（不走完整 agent）
- **请求 Body**：`ProductSearchRequest`
  - `query: string`
  - `limit: number`（默认 5）
  - `image_url?: string`
  - 见 [schemas/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/schemas/agent.py#L31-L34)
- **响应**：`{"status":"success","results":[...]}`（结果结构取决于向量库实现）

#### POST `/api/v1/stream`
- **用途**：SSE 流式输出（打字机效果 + 可附带 products 结构）
- **请求 Body**：同 `ChatRequest`
- **响应**：`text/event-stream`，事件 payload 示例：
  - `{"type":"text","content":"..."}`
  - `{"type":"products","products":[...]}`
  - `{"type":"done"}`
  - `{"type":"error","content":"..."}`
  - 见 [api/agent.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/agent.py#L113-L148)

### 2.4 Rewards API（奖励与财务）

来源：[api/rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/rewards.py)

#### POST `/api/v1/rewards/checkin`
- **用途**：执行每日签到
- **请求 Body**：`{ "user_id": number, "plan_id": string }`
- **响应**：透传 `RewardsService.process_checkin` 结果（成功含 period/day/points_earned 等）
- **错误**：业务失败抛 400（`detail` 为错误原因）

#### GET `/api/v1/rewards/status/{user_id}`
- **用途**：获取用户返现计划进度 + 钱包摘要 + 等级
- **响应**：`{ user_id, wallet, level, plans[] }`
  - `plans[].status` 规范化为手册口径：`active/completed/free_refunded/forfeited`（并保留 `raw_status`）
  - `level.invitees` 为“有效下线数”：两年内至少一笔状态为 `paid/shipped/completed` 且 `refund_status != refunded` 的真实订单用户（避免刷注册/刷退款抬高等级）
  - `level.total_volume` 为“2 年累计分销额”：两年内 `wallet_transactions.type="referral"` 且 `status="completed"` 的金额累计

#### POST `/api/v1/rewards/group-free/verify`
- **用途**：校验 3 人团购免单资格并推进状态
- **请求 Body**：`{ user_id: number, order_id: number, share_code?: string, current_count?: number }`
- **响应**：
  - 成功达成：返回 `status:"success"` 并将 `checkin_plans.status` 置为 `free_refunded`、`group_buy_campaigns.status` 置为 `success`
  - 未达成：返回 `status:"pending"`
- **退款与幂等**：会读取本地 `orders.refund_status` 防重复（`pending/refunded` 会直接跳过），并调用 Shopify Admin API 执行全额原路退款；遇到 429/5xx 会标记 `refund_retry_needed` 并记录错误信息（见 [ledger.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/models/ledger.py) 与 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/rewards.py)）。
- **奖励扣回**：退款成功后会对该订单已发放的奖励做冲正：
  - 分销/粉丝佣金：`wallet_transactions.type="referral"` 生成负向冲正流水（幂等）
  - 签到返现：`wallet_transactions.type="checkin"` 生成负向冲正流水，并同步回退 `checkin_plans.total_earned`（不低于 0）
  - 签到积分：按该订单计划的签到日志日期匹配 `point_transactions.source="SIGN_IN"` 的当日积分发放，并生成负向 `SIGN_IN` 冲正流水（幂等）
  - 续签卡：将该订单计划下未使用的 `renewal_cards` 置为 `revoked`
  见 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/services/rewards.py)。

#### POST `/api/v1/rewards/group-free/retry`
- **用途**：对 `refund_retry_needed` 的订单执行原路退款重试（后台/运营手工触发）
- **请求 Body**：`{ order_id: number, user_id?: number, force?: boolean }`
- **行为**：
  - `refund_status in {pending, refunded}`：直接跳过（幂等）
  - `refund_status == refund_retry_needed`：执行一次 Shopify 退款流程
  - `force=true`：允许对 `failed` 等状态强制重试
- **并发保护**：对 `orders` 行级加锁，避免多次重试并发触发重复退款（见 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/rewards.py)）。

#### POST `/api/v1/rewards/group-free/retry-batch`
- **用途**：批量重试退款（用于定时任务/运营一键修复）
- **请求 Body**：`{ limit?: number, force?: boolean }`
- **选择规则**：
  - 默认仅选择 `orders.refund_status == refund_retry_needed`
  - `force=true` 时会额外选择 `failed`
- **并发保护**：使用 `skip_locked` 行级锁跳过已被其他 worker 选中的订单，防止重复退款（见 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/rewards.py)）。
- **响应**：`{ status:"done", counts:{selected,succeeded,failed,skipped}, results:[...] }`

#### GET `/api/v1/rewards/group-free/retry-queue`
- **用途**：查看待重试队列（只读，用于后台展示/人工点选）
- **Query 参数**：`limit?: number`、`include_failed?: boolean`、`user_id?: number`、`error_status_code?: number`、`cursor?: string`
- **响应**：`{ status:"ok", count:number, items:[{order_id,user_id,refund_status,refund_txn_id,refund_attempts,last_refund_attempt_at,updated_at,error_status_code}] }`

#### POST `/api/v1/webhooks/shopify/orders/paid`
- **用途**：订单支付后触发核心闭环（奖励初始化、推荐分佣、供应链 sourcing、WhatsApp 通知）
- **Header**：`X-Shopify-Hmac-Sha256`（参数名为 `x_shopify_hmac_sha256`）

  - 推荐：`note_attributes[]` 中 `name=="referral_code"` 的 `value`

  - 验签：`verify_shopify_webhook(data, x_shopify_hmac_sha256)`（[webhooks.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/webhooks.py#L18-L38)）

  - 创建签到计划：`RewardsService.init_checkin_plan(customer_id, order_id, reward_base, timezone)`（[webhooks.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/webhooks.py#L90-L98)）
  - 推荐/分佣：`record_referral` + `process_referral_commissions`（[webhooks.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/webhooks.py#L99-L108)）
  - 供应链 sourcing：`SupplyChainService.trigger_sourcing(order_id, line_items)`（[webhooks.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/webhooks.py#L109-L113)）
  - WhatsApp 通知：若 `customer_phone` 存在则发送（[webhooks.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/webhooks.py#L114-L121)）
- **响应**：
  - 正常：`{"status":"ok"}`

### 2.7 Shopify App Proxy（checkin/proxy）

来源：[api/proxy.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/proxy.py)

#### GET `/api/v1/checkin/proxy`
- **用途**：Shopify App Proxy 验签入口（query params 中带 `signature`）
- **Query 参数**：
  - `signature: string`（必填，Shopify 生成）
  - `logged_in_customer_id: string`（常用，用于识别登录用户）
  - 其他 proxy params（原样参与签名计算）
- **验签算法**（当前实现）：
  - 从 query 中 pop 出 `signature`
  - 对剩余参数排序并拼接 `k=v` 形成字符串 `data`
  - `expected_signature = hmac_sha256_hex(SHOPIFY_API_SECRET, data)`
  - `signature != expected_signature` 则 401（[proxy.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/api/proxy.py#L20-L34)）
- **响应**：`{"message":"App Proxy Verified","customer_id":"..."}`（customer_id 为 `logged_in_customer_id`）

## 4. 关系总览（PK/FK 关键链路）

从“订单支付 → 奖励计划 → 钱包/积分 → 供应链 sourcing”角度抽象：

- **用户基座**：`users_ext.customer_id` 是几乎所有“账户体系”的外键根
  - `wallets.user_id`、`wallet_transactions.user_id`
  - `checkin_plans.user_id`、`point_transactions.user_id`、`points.user_id`、`ai_usage_quotas.user_id`
  - `ai_sessions.user_id`
- **签到计划链**：
  - `checkin_plans.order_id` 与 Shopify 订单绑定（paid webhook）
  - `renewal_cards.plan_id -> checkin_plans.id`
  - `checkin_logs.plan_id -> checkin_plans.id`
- **商品链**：
  - `products.supplier_id -> suppliers.id`
  - `product_vectors.product_id -> products.id`
- **推荐链**：
  - `users_ext.inviter_id -> users_ext.customer_id`（自关联）
  - `referral_relationships.inviter_id/invitee_id -> users_ext.customer_id`

### 5.3 媒体云协议 (Media Cloud Protocol v3.1)

1. **唯一来源**：禁止在前端直接引用 `1688.com` 或 `alicdn.com` 的任何媒体资源。
2. **影子重命名**：所有媒体文件在进入 Shopify 云前，必须根据 **Zone 2 影子 ID** 重新命名（格式：`SH_ID_Type_Index`）。目前通过 `Alt Text` 实现逻辑对齐。
3. **分流存储**：
   - **口袋 A (Gallery)**：主图、详情图，通过 `Product.images` 上传。
   - **口袋 B (Legal)**：合规证书图片，上传至 Shopify 文件库，URL 存储在 `0buck_legal.certificates` Metafield。
4. **影子命名规则**：
   - 商品图：`SH_{INTERNAL_ID}_GALLERY_{INDEX}`
   - 证书图：`SH_{INTERNAL_ID}_CERT_{INDEX}`

### 5.5 AI 人格与进化记忆链路 (Persona OS v3.2)

1. **输入阶段**：后端接收到 `user_id` 和消息。
2. **组装阶段 (`assemble_persona_prompt`)**：
   - **L1 强行层**：从 `system_configs` 捞取全局行政指令。
   - **L2 中层**：根据用户选定的模板 ID 从 `persona_templates` 捞取风格描述。
   - **L3 表层**：从 `user_memory_facts` 捞取该用户的 Top 10 高置信度事实。
3. **调用阶段**：将 L1+L2+L3 拼装为 System Prompt，调用 `Gemini-1.5-Pro`。
4. **进化阶段 (Async Reflection)**：
   - 会话结束后，异步启动 `ReflectionService`。
   - 使用 `Gemini-1.5-Flash` 对话进行“激光扫描”事实提取。
   - 更新 `user_memory_facts`，处理冲突并进行时间衰减计算。
5. **审计阶段**：Admin 端通过 `ai_usage_stats` 监控 Token 成本与人格转化率。

### 5.6 Vortex 预判入口协议 (Vortex Predictive Discovery v3.2)

1. **触发时机**：用户进入“发现”频道或打开 2x5 矩阵视图。
2. **后端逻辑 (`get_personalized_discovery`)**：
   - **LTM 检索**：从 `user_memory_facts` 提取 Top 3 核心偏好（如：喜欢蓝色、有只猫）。
   - **向量召回**：以偏好为关键词，从 Qdrant 检索匹配度最高的 10 个商品。
   - **策略排序**：将匹配度最高的商品置于 Index 0，并标记为 `[TRAFFIC]` 策略。
   - **AI 话术合成**：根据用户选定的 L2 人格（可爱萝莉/率直专家）合成个性化欢迎语，提及推荐理由。
3. **前端表现**：
   - **Butler Bubble**：在矩阵顶部弹出气泡显示 AI 话术。
   - **高亮推荐**：Index 0 商品增加 `Vortex_Highlight` 视觉效果（发光边框、专属角标）。
   - **策略透传**：明确显示 `Traffic Deal` 标签，提升点击率。

按优先级：
1. 若 `line_items[].sku` 以 `1688-` 或 `SOURCE-` 开头，则 `source_product_id = sku.split("-", 1)[1]`
2. 否则用 `line_items[].product_id` 拉取 Shopify Product，并扫描 metafields：
   - `namespace == "0buck_sync"` 且 `key in {"source_1688_id","source_id"}`（其中 `source_id` 当前只读不写，属于兼容键）

## 7. 业务计算规则 (Business Logic Rules)

### 7.1 分销等级 (Tiers) 与“有效下线”判定

逻辑实现：见 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/services/rewards.py#L141-L181)

系统根据用户的**分销表现**自动划分等级，等级决定了默认的分销/粉丝奖比例。

| 等级 | 英文名 | 默认佣金比例 | 晋级条件（满足其一） |
| :--- | :--- | :--- | :--- |
| **白银** | `silver` | 1.5% | 默认初始等级 |
| **黄金** | `gold` | 2.0% | **5 个有效下线** 或 **$500 累计分销额** |
| **铂金** | `platinum` | 3.0% | **20 个有效下线** 或 **$2,000 累计分销额** |

**有效下线 (Effective Downline) 判定标准**：
- 统计范围：**两年内**（滚动窗口）建立推荐关系的下线。
- 激活条件：该下线用户在两年内至少产生一笔符合以下条件的订单：
  - **订单状态**：`status` 为 `paid`（已支付）、`shipped`（已发货）或 `completed`（已完成）。
  - **退款状态**：`refund_status` 不为 `refunded`（已退款）。
- 目的：杜绝虚假注册（空关系）或刷单退款套取高佣金比例。

### 7.2 奖励扣回与冲正 (Clawback)

当发生“团购免单退款”或“恶意订单退款”时，系统执行全量奖励回收。详见 [rewards.py](file:///Volumes/SAMSUNG%20970/AccioWork/coder/0buck/backend/app/services/rewards.py) 中的 `clawback_rewards_for_order`。
